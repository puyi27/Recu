1. Imágenes, Logo SVG y FontAwesome (Ej. 1)

Logotipo SVG:

<svg width="200" height="60" viewBox="0 0 200 60">
  <text x="50%" y="50%" fill="#fff" text-anchor="middle">MI LOGO SVG</text>
</svg>
💡 Qué es: El atributo viewBox define el sistema de coordenadas. La etiqueta <text> usa x="50%" y="50%" junto con text-anchor="middle" y dominant-baseline="middle" para centrar el texto matemáticamente en el centro del SVG, independientemente del tamaño de la pantalla.


Imagen en Promoción (Responsiva):

<picture>
  <source media="(min-width: 576px)" srcset="img-1000px.jpg">
  <img src="img-500px.jpg" alt="Producto Responsivo">
</picture>
💡 Qué es: La etiqueta <picture> permite Art Direction (cambiar la imagen por completo según la pantalla). El <source> define una condición (ej. pantallas anchas). Si se cumple, usa ese archivo. Si no, o si el navegador no soporta picture, usa la imagen por defecto del <img>. Ideal para optimizar el rendimiento y no cargar imágenes gigantes en móviles.

2. Audio y Vídeo HTML5 con API (Ej. 1)

Audio (Múltiples formatos para compatibilidad):

<audio controls preload="metadata">
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
</audio>
💡 Qué es: controls muestra el reproductor nativo del navegador. Ponemos varios <source> para asegurar que el navegador reproduzca el primer formato que entienda (MP3, OGG, WAV). preload="metadata" solo descarga la duración y los datos básicos inicialmente, ahorrando datos al usuario.


Vídeo con Poster, Silenciado y Controles API JS:

<video id="miVideo" poster="poster.jpg" muted>
  <source src="video.mp4" type="video/mp4">
</video>
<button onclick="document.getElementById('miVideo').play()">Play</button>
<script>
  function modificarVolumen(cambio) {
      const vid = document.getElementById('miVideo');
      vid.volume = Math.max(0, Math.min(1, vid.volume + cambio));
  }
</script>
💡 Qué es: El atributo poster muestra una imagen antes de darle a Play. muted es obligatorio en muchos navegadores para permitir que se reproduzca automáticamente. Las funciones JS usan el API Multimedia HTML5: .play(), .pause() y .volume (que va obligatoriamente de 0.0 a 1.0).

3. Transiciones y Transformaciones CSS3 (Ej. 2)

A) Menú con transición (Efecto Cortina de arriba a abajo):

Inicio Productos Contacto 
<style>
  .nav-item::before {
      content: ""; position: absolute; top: -100%; left: 0;
      width: 100%; height: 100%; transition: top 0.3s ease;
  }
  .nav-item:hover::before { top: 0; }
</style>
💡 Qué es: Una transición clásica de efecto cortina. Usamos ::before para crear un pseudoelemento de color oculto justo arriba (top: -100%). Al hacer :hover en el botón, le decimos que baje a top: 0. La regla transition: top 0.3s hace que este movimiento sea suave.

B) Transformaciones al pasar el ratón (Hover):

transform: scale()

transform: translateY()

<style>
  .hover-escala:hover { transform: scale(1.1); }
  .hover-translate:hover { transform: translateY(-10px); }
</style>
💡 Qué es: transform: scale(1.05) amplía el elemento un 5% desde su centro. transform: translateY(-8px) lo desplaza físicamente 8 píxeles hacia arriba. Acompañado de un transition: transform en el estado base, genera microinteracciones fluidas muy profesionales.

4. Animaciones Web: CSS, Canvas y SVG (Ej. 3)

1. HTML5 y CSS3 (@keyframes) - Slider Ininterrumpido:

<style>
  .slider-mov { display: flex; width: 300%; animation: moverSlider 10s infinite; }
  @keyframes moverSlider {
      0%, 25% { transform: translateX(0); }
      33%, 58% { transform: translateX(-33.33%); }
      66%, 91% { transform: translateX(-66.66%); }
      100% { transform: translateX(0); }
  }
</style>
💡 Qué es: El contenedor interno (.slider-mov) mide un 300% de ancho porque caben 3 imágenes en fila. La animación @keyframes desplaza el contenedor en el eje X: en 33% se desplaza a la segunda imagen (-33.3%), en 66% a la tercera, y al final vuelve a 0. El infinite hace que no se detenga nunca.

Extra Temario: Pausar/Reanudar animación con JS:

<script>
  $("#btn-pausa-css").click(() => $("#pendulo").css("animation-play-state", "paused"));
  $("#btn-play-css").click(() => $("#pendulo").css("animation-play-state", "running"));
</script>
💡 Qué es: Se puede pausar y reanudar cualquier animación CSS pura dinámicamente usando JS o jQuery cambiando la propiedad animation-play-state entre running y paused.

2. Animación con API Canvas (Barra de Likes 0 a 200):


<canvas id="canvasLikes" width="400" height="50"></canvas>
<script>
  const ctx = document.getElementById("canvasLikes").getContext("2d");
  let likes = 0;
  function dibujar() {
      ctx.clearRect(0, 0, 400, 50);
      ctx.fillRect(0, 0, (likes / 200) * 400, 50); 
  }
</script>
💡 Explicación Detallada (Examen):

El API Canvas es como un lienzo de pintura. Funciona por "fotogramas", así que para actualizar algo, la regla de oro es: Borrar y volver a Dibujar.

  • getContext("2d"): Obligatorio. Le dice al navegador que vas a dibujar en plano (X, Y). Te devuelve un objeto (ctx) con todas las herramientas (pinceles).
  • clearRect(0, 0, 400, 50): Borra un rectángulo entero desde la coordenada 0,0 hasta el final del canvas. Es vital llamarlo en cada actualización, de lo contrario, pintarías encima del trazo anterior.
  • fillRect(x, y, ancho, alto): Dibuja un rectángulo pintado. En los exámenes, la clave está en el ancho dinámico. Usamos una Regla de 3: (likes / max_likes) * ancho_total_canvas.

3. Animación SVG: Dos imágenes persiguiéndose en trayectoria:

<svg width="100%" height="250">
  <path id="rutaSVG" d="M 50 200 Q 250 10 450 200" fill="none"/>
  <image href="img1.png">
    <animateMotion dur="5s" repeatCount="indefinite" rotate="auto">
      <mpath href="#rutaSVG"/>
    </animateMotion>
  </image>
  <image href="img2.png">
    <animateMotion dur="5s" begin="0.8s" repeatCount="indefinite" rotate="auto">
      <mpath href="#rutaSVG"/>
    </animateMotion>
  </image>
</svg>
💡 Explicación Detallada (Examen):

Las animaciones SVG se componen del camino (<path>) y el elemento que se mueve (<animateMotion>).

  • El atributo d="..." (Data del Path): Define matemáticamente el trazado.
    • M 50 200: (Move To) Mueve el pincel a la coordenada X:50, Y:200 sin pintar. Es el punto de inicio.
    • Q 250 10 450 200: (Quadratic Bezier) Dibuja una curva. Los primeros dos números (250 10) son el "imán" invisible que tira de la línea hacia arriba. Los últimos dos (450 200) son el punto final de esta curva.
    • T 850 200: (Smooth Quadratic) Continúa la curva anterior con una onda suave hasta la coordenada 850 200.
  • <animateMotion> y <mpath>: Se incrusta dentro de la imagen que quieres mover. La etiqueta interna mpath href="#rutaSVG" vincula la animación con el camino dibujado arriba.
  • El truco de la persecución: Al poner begin="0.8s" en la segunda imagen, forzamos un retraso. Esta arranca casi 1 segundo tarde, dando el efecto visual de "persecución".

5. Extra Examen: Formulario Dinámico y Drag & Drop API

A) Arrastra el producto a la cesta de compra (Drag & Drop):

Producto Especial
Cesta de Compra 
<div id="producto" draggable="true" ondragstart="arrastrar(event)">Caja</div>
<div id="zona" ondragover="permitir(event)" ondrop="soltar(event)"></div>
<script>
  function arrastrar(e) { e.dataTransfer.setData("id", e.target.id); }
  function permitir(e) { e.preventDefault(); } // MUY IMPORTANTE
  function soltar(e) {
      e.preventDefault();
      let id = e.dataTransfer.getData("id");
      e.target.appendChild(document.getElementById(id));
  }
</script>
💡 Explicación Detallada (Examen):

El API Drag & Drop de HTML5 tiene 3 fases críticas que siempre debes programar:

  • Fase 1 (Inicio - ondragstart): El elemento HTML necesita obligatoriamente draggable="true". En el evento, usamos e.dataTransfer.setData("id", e.target.id). Esto es como un portapapeles temporal: guarda el ID de la caja que estás agarrando.
  • Fase 2 (¡LA TRAMPA! - ondragover): Por seguridad, los navegadores prohíben soltar cosas encima de otras. En la zona de destino, tienes que atrapar el evento ondragover y escribir e.preventDefault() para anular esa prohibición. Si olvidas esto, el bloque no se soltará nunca.
  • Fase 3 (Soltar - ondrop): En la zona de destino, recuperamos el ID del portapapeles con e.dataTransfer.getData("id"). Por último, usamos el DOM (appendChild) para mover físicamente el elemento HTML de un lado a otro.

B) Lista de Tareas Dinámica (Manipulación DOM jQuery):

  • Estudiar DOM jQuery 
<script>
  // 1. Añadir al DOM
  $("#btn-add").click(function() {
      let valor = $("#input").val();
      $("#lista").append(`<li>${valor} <button class="del">X</button></li>`);
  });

  // 2. Delegación de eventos (borrar elementos generados dinámicamente)
  $("#lista").on("click", ".del", function() {
      $(this).parent().remove();
  });
</script>
💡 Qué es: Manipulación del DOM con jQuery. .append() inserta HTML al final. ¡OJO EXAMEN! Como los botones de borrar se crean dinámicamente (no existen al cargar la página), NO puedes usar $(".del").click() directo. Debes usar "Delegación de Eventos": $("#lista").on("click", ".del", fn) que asigna la escucha al padre estático que siempre existió.

C) Web Storage API (Guardar datos en el navegador):

<script>
  // Guardar
  $("#btn-guardar").click(() => localStorage.setItem("usuario", $("#nombre-user").val()));
  
  // Recuperar
  $("#btn-cargar").click(function() {
      let nom = localStorage.getItem("usuario");

</script>
💡 Qué es: localStorage permite guardar datos permanentemente en el navegador. No se borran al cerrar la pestaña (se usa mucho para guardar el Modo Oscuro o un login). Sus dos funciones principales son setItem("clave", valor) y getItem("clave").

D) Canvas Avanzado: Gráfico Circular (Pie Chart):

50% Aprobados (Web)

50% Suspensos (Logos)

<canvas id="canvasPie" width="200" height="200"></canvas>
<script>
  const ctxP = document.getElementById("canvasPie").getContext("2d");
  
  // arc(x, y, radio, anguloInicio, anguloFin)
  // Nota: Los ángulos usan Radianes (Math.PI * 2 = 360º)
  
  ctxP.beginPath();
  ctxP.moveTo(100, 100);
  ctxP.arc(100, 100, 100, 0, Math.PI, false); // 0 a 180º
  ctxP.fillStyle = "#3498db";
  ctxP.fill();

  ctxP.beginPath();
  ctxP.moveTo(100, 100);
  ctxP.arc(100, 100, 100, Math.PI, Math.PI * 2, false); // 180º a 360º
  ctxP.fillStyle = "#e74c3c";
  ctxP.fill();
</script>
💡 Qué es: Mientras que hacer rectángulos es fácil (fillRect), dibujar gráficas requiere arc(). El truco es usar beginPath(), mover el "lápiz" al centro exacto con moveTo(), trazar el arco definiendo los ángulos en Radianes (Math.PI es media vuelta completa), y pintarlo con fill().
