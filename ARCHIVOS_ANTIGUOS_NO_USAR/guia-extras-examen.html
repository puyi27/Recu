<style>
  .grid-examen {
      display: grid;
      /* TRUCO EXAMEN: auto-fit crea tantas columnas de 250px como quepan. 
         Si la pantalla es pequeña (móvil), bajan a la siguiente línea automáticamente. */
      grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
      gap: 20px;
  }
</style>
💡 Explicación Detallada (Examen):

En lugar de escribir pesadas Media Queries (@media (max-width:...)) para hacer que el diseño sea responsive, CSS Grid tiene una fórmula mágica:

  • repeat(...): Le dice al navegador que repita la creación de columnas automáticamente.
  • auto-fit: Hace que el grid calcule cuántas columnas caben en la pantalla actual. Si la pantalla es gigante, pondrá muchas; si es un móvil, pondrá 1 sola.
  • minmax(250px, 1fr): Es la regla de tamaño. La columna nunca será más estrecha de 250px. Si hay espacio extra, se repartirá equitativamente (1fr = 1 fracción libre).

2. Carga Dinámica de Datos (Fetch API / AJAX)

Cargar un JSON externo y pintarlo en la página:

    <script>
  // Usando el moderno Fetch API (Recomendado y nativo en Javascript)
  $("#btn-cargar-datos").click(function() {
      fetch('https://jsonplaceholder.typicode.com/todos?_limit=3')
        .then(response => response.json()) // Lo convertimos a formato JSON
        .then(datos => {
            $("#lista-api").empty(); // Vaciar la lista antes de añadir
            datos.forEach(tarea => {
                // Dibujamos cada dato que viene del servidor usando Template Literals (`)
                $("#lista-api").append(`<li>[ID: ${tarea.id}] ${tarea.title}</li>`);
            });
        })
        .catch(error => console.error("Error al cargar:", error));
  });
</script>
    💡 Explicación Detallada (Examen):

    El API Fetch es el estándar moderno en Javascript para conectarse a servidores externos o cargar archivos .json locales. Es asíncrono y usa Promesas (.then()).

    • fetch('URL'): Realiza la petición de red para traer los datos.
    • .then(response => response.json()): Paso OBLIGATORIO. El servidor te devuelve un texto crudo. Esta línea lo parsea y lo convierte en un Objeto/Array real de Javascript.
    • .then(datos => { ... }): Aquí ya tienes los datos listos. Normalmente, se usa datos.forEach() para iterar sobre la lista y hacer un .append() en el HTML.

    3. Semántica y Accesibilidad (Puntos Fáciles)

    A) Estructura Semántica HTML5 (¡Ojo con usar solo divs!):

    <body>
  <header> <!-- Cabecera (Logos, menú) -->
      <nav>...</nav> <!-- Menú de navegación principal -->
  </header>
  
  <main> <!-- Contenido principal de la página (OBLIGATORIO) -->
      <section> <!-- Sección temática -->
          <article>...</article> <!-- Contenido independiente (ej. Noticia, Producto) -->
      </section>
      <aside>...</aside> <!-- Barra lateral (publicidad, menús secundarios) -->
  </main>
  
  <footer>...</footer> <!-- Pie de página (Legal, copyright, enlaces) -->
</body>
    💡 Explicación Detallada (Examen):

    En diseño web, usar únicamente <div> para todo está penalizado. Los lectores de pantalla para ciegos necesitan entender la estructura de la página usando estas etiquetas (Semántica HTML5):

    • <header> y <footer>: Definen el principio y el final.
    • <main>: Todo el contenido vital debe ir dentro de esto. Solo puede haber uno por página.
    • <section> y <article>: Reemplazan a los divs grandes. Article se usa si el contenido tiene sentido por sí solo (ej. un producto, un post del blog).

    B) Etiquetas ARIA y Accesibilidad de Teclado (Eventos Combinados):

    Clica aquí o púlsame con Tabulador + Enter 
    <!-- 1. tabindex="0" permite hacer focus con el teclado a cosas que no son botones o links nativos -->
<!-- 2. aria-label="" se usa cuando hay un icono sin texto para que el lector de pantalla sepa qué es -->
<!-- 3. role="button" avisa a los lectores de que este div actúa como botón -->
<div tabindex="0" aria-label="Cerrar ventana" role="button" id="btn-cerrar"> <i class="fas fa-times"></i> </div>

<script>
  // Evento mixto (Click de ratón O pulsación de teclado)
  $("#btn-teclado").on("click keypress", function(e) {
      // Si fue click (e.type) O fue pulsación de tecla Enter (e.key)
      if(e.type === "click" || e.key === "Enter") {
          alert("¡Acción lanzada de forma 100% accesible!");
      }
  });
</script>
    💡 Explicación Detallada (Examen):

    Si creas un botón falso usando un <div>, el usuario de teclado no puede interactuar con él (el tabulador lo ignorará). Hay que arreglar 3 cosas:

    • tabindex="0": Fuerza al navegador a permitir que este elemento reciba foco con la tecla Tabulador.
    • role="button" y aria-label: Le chiva al lector de voz que eso no es un texto normal, sino un botón, y le dice qué hace.
    • Evento Mixto de Teclado: Usamos on("click keypress") para detectar ambas acciones. Dentro, leemos e.key === "Enter" para ejecutar el código cuando se pulsa el intro.

    4. Ejercicios Prácticos Clásicos de Examen

    A) Acordeón (Mostrar uno, ocultar los demás):

    + Sección 1
    Contenido oculto de la sección 1. Aquí se usa slideToggle().
    + Sección 2
    Contenido oculto de la sección 2. Al abrir este, el otro se cierra automáticamente.
    <script>
  $(".acc-titulo").click(function() {
      // 1. Ocultar todos los contenidos abiertos con animación
      $(".acc-contenido").slideUp();
      
      // 2. Si el que he clicado estaba cerrado, lo abro
      if(!$(this).next().is(":visible")) {
          $(this).next().slideDown();
      }
  });
</script>
    💡 Por qué apuntarlo: Es el ejercicio estrella de DOM con jQuery. El truco de examen está en usar $(this).next() para seleccionar el contenido que está justo debajo del título al que hiciste clic, sin necesidad de ponerles IDs diferentes a cada uno.

    B) Ventana Modal Emergente:

    ¡Aviso Importante!

    Has abierto la ventana modal.

    <script>
  // Abrir modal (Cambiamos de display:none a display:flex)
  $("#btn-abrir-modal").click(() => $("#mi-modal").css("display", "flex"));
  
  // Cerrar desde el botón
  $("#btn-cerrar-modal").click(() => $("#mi-modal").hide());
  
  // Cerrar al hacer clic fuera (en el fondo oscuro)
  $("#mi-modal").click(function(e) {
      if(e.target.id === "mi-modal") $(this).hide();
  });
</script>
    💡 Por qué apuntarlo: Te evalúan si sabes usar CSS position: fixed para cubrir la pantalla completa, y si sabes atrapar eventos para cerrarla. ¡Punto extra de examen! El e.target.id === "mi-modal" evita que la ventana se cierre incorrectamente si haces clic dentro del cuadrado blanco central.

    C) Intercambio de Atributos (Galería de miniaturas):

    <script>
  $(".miniatura").click(function() {
      // 1. Coger el src de la miniatura clickada
      let nuevoSrc = $(this).attr("src");
      
      // 2. Ponérselo a la imagen grande principal
      $("#img-principal").attr("src", nuevoSrc);
      
      // 3. Cambiar el borde activo visualmente
      $(".miniatura").css("border-color", "transparent");
      $(this).css("border-color", "var(--accent-color)");
  });
</script>
    💡 Por qué apuntarlo: Evalúa si sabes leer y escribir atributos HTML en tiempo real con .attr(). En simulacros de tiendas online de exámenes (ej. "cambiar el color de una camiseta al pulsar un botón"), esto es exactamente lo que esperan que programes.

    D) Botón "Volver Arriba" (Eventos de Scroll):

    Haz scroll hacia arriba y abajo en esta página. Fíjate en el botón flotante rojo que aparece abajo a la derecha de tu pantalla. 
    <script>
  // Escuchar el evento scroll de la ventana general
  $(window).scroll(function() {
      // Si bajamos más de 300px, mostramos el botón
      if ($(this).scrollTop() > 300) {
          $("#btn-up").fadeIn();
      } else {
          $("#btn-up").fadeOut();
      }
  });

  // Al hacer clic, subir suavizado
  $("#btn-up").click(function() {
      // Javascript nativo moderno para subir suave (Recomendado sobre jQuery animate)
      window.scrollTo({ top: 0, behavior: 'smooth' });
  });
</script>
    💡 Por qué apuntarlo: Las funciones $(window).scroll() y scrollTop() te pueden caer para hacer barras de progreso laterales, menús pegajosos o botones de "Volver arriba". Es un ejercicio clásico que mezcla eventos globales de la ventana (no de un botón) y animaciones de JS moderno (behavior: 'smooth').
